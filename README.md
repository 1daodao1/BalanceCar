# BalanceCar - 基于 STM32 的双轮自平衡小车

## 项目简介
这是一个基于 STM32F1 系列微控制器（使用 HAL 库）和 FreeRTOS 实时操作系统的双轮平衡小车项目。本项目通过读取 MPU6050 传感器数据进行姿态解算，并结合带有编码器的直流减速电机，实现了小车的自平衡、速度控制以及转向控制。同时支持通过蓝牙模块进行远程遥控和 PID 参数在线调试。

## 硬件架构
- **主控芯片**: STM32F1xx (如 STM32F103)
- **姿态传感器**: MPU6050 (六轴加速度计和陀螺仪)
- **执行机构**: 带编码器的直流减速电机 + 电机驱动模块
  - 编码器接口: 使用 TIM3 和 TIM4 的编码器模式
  - PWM 控制: 使用 TIM2 输出 PWM 驱动电机
- **显示模块**: OLED 屏幕，用于实时显示系统状态
- **通信模块**: 
  - 蓝牙模块 (USART2, 9600 波特率)
  - 串口调试 (USART1, 115200 波特率)
- **人机交互**: 板载按键与 LED 指示灯

## 软件架构
项目基于 STM32CubeMX 生成底层初始化代码，并搭载 **FreeRTOS** 进行多任务调度。核心业务划分为以下四个任务：

1. **直立任务 (Balance Task)**
   - 优先级：`Realtime`
   - 运行周期：5ms (由 TIM1 定时器中断触发)
   - 功能：读取 MPU6050 数据，利用**互补滤波**算法计算小车倾角；执行直立环 (Angle) PID 控制，计算并输出控制电机运行的基础 PWM。
2. **速度任务 (Speed Task)**
   - 优先级：`High`
   - 运行周期：50ms
   - 功能：读取左右轮编码器脉冲，计算实际速度；执行速度环 (Speed) 和转向环 (Turn) PID 控制。速度环的输出作为直立环的目标角度（串级 PID）。
3. **通信任务 (Comms Task)**
   - 优先级：`Normal`
   - 运行周期：20ms
   - 功能：解析蓝牙串口接收到的数据包，支持手机 APP 遥控器摇杆控制、PID 参数在线调整，并将实时数据打包发送至上位机进行波形显示。
4. **UI 任务 (UI Task)**
   - 优先级：`Low`
   - 运行周期：100ms
   - 功能：定时刷新 OLED 屏幕，显示当前的角度、各环 PID 参数、目标与实际速度等运行状态。

## 控制算法
* **姿态解算 (互补滤波)**: 融合 MPU6050 的加速度计和陀螺仪数据，过滤高频和低频噪声，获取稳定准确的俯仰角。
* **串级 PID 控制**:
  * **直立环 (Angle PID)**: 保持小车平衡，根据当前角度和目标角度输出控制量。使用了微分先行(Derivative on Measurement)策略以提高稳定性。
  * **速度环 (Speed PID)**: 控制小车前后运动，输出作为直立环的目标角度（限幅）。
  * **转向环 (Turn PID)**: 根据目标转向差速控制左右电机差速，实现小车转向。

## 目录结构说明
* `Core/`: STM32CubeMX 生成的核心代码 (main.c, freertos.c 等)
* `Hardware/`: 外设驱动层代码 (OLED, MPU6050, 编码器, 蓝牙, 按键, LED 等)
* `User/`: 用户应用层代码 (用户任务 user_task.c，PID 控制 PID.c)
* `Drivers/`: STM32 HAL 库和 CMSIS 库
* `Middlewares/`: FreeRTOS 源码
